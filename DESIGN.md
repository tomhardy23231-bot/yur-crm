# DESIGN.md — ЮрКейс v1.0

> **Источник правды UI.** Любой UI-код (шрифт, цвет, отступ, радиус, компонент)
> читает этот файл ПРЕЖДЕ, чем выбрать значение. Отклонения — только с явным
> согласованием.
>
> **Действующая система — редизайн 2026-06-03:** строгий светлый вид, ОДИН
> синий акцент `#2563EB`, тёмный ink-сайдбар, белые карточки на холодном
> сером paper-фоне. Темы TEAL и «латунь» УДАЛЕНЫ (история — в git).
> **Ревизия 2026-06-12 (v3 Сессия 10):** документ приведён в соответствие
> коду (`src/app/globals.css` — фактический реестр токенов), добавлены
> AA-контрастные `*-fg`/`*-text` тона, токен оверлея, компонентные алиасы
> радиусов.
>
> **Редизайн 2026-06-12 (Волна 0 — фундамент):** paper-фон облегчён
> `#C6D1DF → #F1F4F9` (карточки больше не «парят» на тяжёлом сером);
> `--shadow-sm` срезана до одного тонкого слоя; поля ввода (`Input` /
> `Textarea` / `Select`) — белый фон + видимая граница + синее фокус-кольцо
> вместо серой заливки без бордера («1С-look»). Срез ALL-CAPS-подписей,
> тон цвета и структура детальных экранов — по экранам в следующих волнах.

## 1. Контекст

- **Что:** CRM для юридической компании; внутренний инструмент на 8 часов в день.
- **Кто:** owner, admin, офис-менеджеры, юристы (продажники), эксперты (исполнители).
- **Класс:** data-tool — плотные таблицы, списки «карточки-строки», формы, kanban-воронка, отчёты.
- **Язык интерфейса:** украинский (дефолт) + русский.
- **Характер:** строгий, корпоративный, спокойный. Цвет работает на семантику
  (этап / категория / приоритет / деньги), хром нейтрален. Не «1С», не generic-админка.

## 2. Типографика

| Роль | Шрифт | Подключение | Применение |
|---|---|---|---|
| UI / body / display / **цифры** | **IBM Plex Sans** (400 / 500 / 600 / 700) | `next/font`, subsets `latin + cyrillic` | Всё: заголовки, тело, кнопки, таблицы, суммы |
| Клавишные подсказки | **JetBrains Mono** (400 / 600 — только эти веса) | `next/font`, subset `latin` | ТОЛЬКО `<kbd>`-подсказки и чип прогресса тура |

  перекрывают.
- Базовый размер 14px / 1.55. Заголовки страниц — в топбаре (17px bold), не на страницах.

### Запреты
- ❌ Inter, Roboto, Arial, Helvetica, system-ui для основных ролей.
- ❌ Serif в display (Lora, Newsreader, Fraunces — мимо).
- ❌ Mono для сумм/номеров в таблицах (только kbd).

## 5. Тени и движение

Тени: `--shadow-sm` (статичные карточки) → `--shadow-md` (hover/popover) →
`--shadow-lg` → `--shadow-pop` (модалки). Hover-тени кнопок —
`--shadow-primary-hover` / `--shadow-destructive-hover` (color-mix от токена —
следуют за `--primary`/`--error`).

Движение: `--ease-out/in/inout`; `--dur-micro 80ms / short 180ms / medium 320ms`.
Появления: `fade-in-up`, `screen-in`, `sheet-*`, `bar-grow`; валидация — `shake`;
сворачивание сайдбара — `t-resize`. Всё гасится `prefers-reduced-motion`.

**Правила (решения пользователя 2026-06-07, сохранять):**
- ❌ НИКАКИХ infinite-анимаций в длинных списках (пульс этапа дёргал скролл) —
  у `StageBadge` есть проп `pulse={false}` для списков.
- ❌ Hover строк/карточек списков — БЕЗ `translate-y` (только цвет/тень).
- ✅ Lift (`-translate-y-px` + цветная тень) — только у CTA-кнопок (primary/destructive).

## 6. Компоненты

Архитектура shadcn/ui (CVA + Radix + Tailwind), имена токенов наши. База:

- **Button** — `primary` (сплошной синий, hover-lift), `secondary` (белая+бордер),
  `ghost`, `destructive`; высоты 36/32/40, `rounded-md` (=9px control).
- **Input / Textarea / Select** (редизайн 2026-06-12 Волна 0) — белое поле
  (`bg-surface`) + видимая граница (`border-border`, hover → `border-strong`),
  фокус — синяя граница + кольцо (`border-primary` + `ring-primary-subtle`).
  Серую заливку без бордера (`bg-surface-muted border-transparent`) НЕ
  использовать — это «1С-look». Граница инпута — мягкий хэйрлайн (< 3:1 по
  1.4.11, как и было); критичный контраст несёт фокус-кольцо (5.17:1).
- **Badge** — pill (`rounded-full`), точка+текст; тона semantic/prio; текст
  залитых — `*-text` тона. Вариант `quiet` — цветная точка + тёмный текст без
  заливки (для плотных таблиц).
- **StageBadge / CategoryBadge** — единый язык: чип `rounded-chip` с круглой
  точкой, подложка `*-bg` + текст `*-fg`. Различие этапа и категории — цветовой
  гаммой (этапы — холодная шкала, категории — cyan/pink/magenta) и контекстом
  колонок, НЕ формой. `quiet`-вариант — точка + тёмный текст. Пульс точки —
  только у «живого» этапа и не в списках.
- **Карточки-строки списков** (`CardListShell` + `ClickableCard`) — десктоп-вид
  списков дел/клиентов: каждая строка — карточка на paper-фоне; шапка колонок
  липкая (`sticky top-0`, фон `--bg`), подписи `text-text-muted` 10px caps.
  Горизонтальный скролл узких экранов даёт контент-зона app-shell (внутри
  компонента overflow не заводить — сломает sticky).
- **Table** — плотная: ряд 44px (`h-11`), шапка 40px caps `text-text-subtle`
  (на белом — AA), `sticky` thead, `border-separate` + границы на ячейках
  (границы едут вместе со sticky-шапкой). Числа — вправо, `tabular-nums`.
- **Modal / ConfirmDialog** — `rounded-modal`, подложка `bg-overlay` +
  backdrop-blur, фокус-трап, Esc; деструктив — красная CTA.
- **Avatar** — круг (или `square` 8px для таблиц), инициалы на `--avatar-N`.
- **Командная палитра** `Cmd/Ctrl+K`; kbd-подсказки — JetBrains Mono.
- **Toast** (`ui/toast.tsx`, v3 s11) — обратная связь на действие.
  `useToast().success/error`; для форм, чьи actions делают redirect
  (client-form/case-form) — `flashToast()` через sessionStorage (TTL 15 с,
  показывает провайдер на новой странице). Позиция: desktop — правый низ
  (`bottom-4 right-4`), mobile — по центру над bottom-nav; `z-[110]` (выше
  модалок). Карточка: `bg-surface` + бордер + `rounded-card` + `--shadow-lg`,
  иконка 16px (`CheckCircle2`/`AlertCircle` в success/error-тонах), текст 13px.
  Максимум 3 одновременно, auto-dismiss 4 с с паузой при hover. Появление —
  `animate-toast-in` (fade+slide 150ms, гасится reduced-motion). A11y:
  контейнер `aria-live="polite"`; success → `role="status"`, error → `"alert"`.
  Когда что: success — создание/сохранение сущности (платёж, акт, задача,
  отпуск, «Сохранено» после redirect-форм); error — провал action'а, где нет
  инлайн-баннера формы. `?error=`-баннеры страниц тосты НЕ заменяют.
- **EmptyState** (`ui/empty-state.tsx`, v3 s11) — единый вид «пусто»:
  опц. иконка 28px `text-text-subtle`, title 14px medium, hint 13px muted,
  опц. CTA. Без собственной рамки — родитель оборачивает (Card-секция как
  есть; страницы списков — карточка `rounded-lg border bg-surface py-8`).
  У списков ДВА текста: «пусто вообще» (с CTA) и «не найдено по фильтрам».
- **Горячие клавиши** (`app/hotkeys-provider.tsx`, v3 s11): `Ctrl+K` и `/` —
  палитра; `N` — новое дело (только с правом `create_cases`); `T` — новая
  задача (`/tasks?new=1`); `?` — модал-шпаргалка; `Esc` — закрыть.
  Сравнение по `e.code` (работает на укр/рус раскладке); игнор внутри
  полей ввода, contenteditable, открытых `role="dialog"` и при Ctrl/Meta/Alt.
  Список дублируется в `/help` («Горячие клавиши») и футере палитры.
- **Быстрые фильтры** (`cases/quick-filters.tsx`, v3 s11) — чипы-ссылки над
  рядом фильтров /cases: «С долгом» (`?debt=true`), «Закрытые за месяц»
  (архив + closed_from/to текущего киевского месяца), «Зависшие»
  (`?sort=stage_changed_at&dir=asc`). Чип `rounded-chip` h-8; активный —
  `bg-primary-subtle text-primary border-primary-border` (= вкладки), клик по
  активному сбрасывает на `/cases`. Пресет = только параметры, которые
  страница уже парсит; новых фильтров в queries не заводить.
- **Состояния** — обязательны: empty-state (`EmptyState`), skeleton
  (`ListingSkeleton` и др.), error-границы (`global-error`, `(app)/error`,
  `not-found`), `?error=`-баннеры, shake-валидация форм, toast-обратная связь.

## 7. Доступность (WCAG AA — обязательна)

- Обычный текст ≥ 4.5:1, крупный/декоративный ≥ 3.0:1. Проверять скриптом
  (PLAN-V3 Сессия 10 — `contrast.js`), не на глаз.
- Залитые чипы/бейджи — ВСЕГДА пара «подложка `*-bg` + тёмный текст
  `*-fg`/`*-text`». Яркий тон на подложке — только точка/иконка.
- На paper-фоне (`--bg`) вторичный текст — `text-text-muted`
  (`text-text-subtle` там не проходит).
- Фокус видим всегда: `outline 2px var(--primary)` (в сайдбаре —
  `--sidebar-accent-bright`).
- Цвет — не единственный носитель смысла: этап/категория/приоритет дублируются
  текстом, статусы — подписями.
- Клавиатура: все интерактивы фокусируемы, `aria-label` у кнопок-иконок,
  модалки с фокус-трапом и Esc.
