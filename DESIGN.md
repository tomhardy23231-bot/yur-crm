## 1. Контекст

- **Что:** CRM для юридической компании.
- **Кто:** владелец, администраторы, офис-менеджеры, юристы (продажники), эксперты (исполнители) — 5 ролей.
- **Где:** внутренний рабочий инструмент, 8 часов в день, RU-интерфейс.
- **Класс:** data-tool (плотные таблицы, формы, документы, kanban-воронка).
- **Архетип:** **«ЮрКейс» — премиальная юридическая CRM.** Тёмный ink-сайдбар +
  светлая paper-зона, латунный (legal gold) акцент, мягкие тени и скругления,
  моноширинные цифры. Строгий, но живой — не «1С» и не generic-админка.

---

## 2. Memorable thing

> **«Современный, удобный, прозрачный».** Светлый интерфейс с цветными визуальными
> якорями (этап дела, приоритет, источник коммуникации) — сотрудник за секунду
> видит «где это дело, насколько горит, как с клиентом связались».

---

## 3. Визуальное направление

- **Тон:** строгий, премиальный, но живой. Бренд — **латунь / legal gold (`#B88A3E`)**,
  латунный градиент (`#B88A3E → #8F6A2C`) на CTA-кнопках, бренд-плашке и hero-шапках
  карточек.
- **База:** «paper» `#EEF1F6` (рабочая зона), белые карточки, тонкие границы `#E3E8EF`,
  мягкие объёмные тени, скругления до `22px`.
- **Сайдбар:** «ink» — тёмный градиент (`#232834 → #1C212C`), латунный акцент активного
  пункта.
- **Семантика денег:** зелёный «money» `#1F8A5B` (оплачено/успех), красный «debt»
  `#CF4242` (задолженность/удаление).
- **Reference:** кликабельный прототип «ЮрКейс» (эталон заказчика).
- **Anti-reference:** «1С-look», generic shadcn (slate-default), индиго-фиолетовые
  градиенты на белом, refined-minimal без цвета.

---

## 4. Тёмная тема

**Рабочая зона (контент) — только светлая.** Полноценная тёмная тема не
планируется; не закладывать `.dark`-вариантов и переменных «на потом».

**Исключение (решение 2026-05-29): навигация (sidebar) — тёмная.** Это
ЕДИНСТВЕННАЯ тёмная поверхность («тёмный сайдбар + светлая рабочая зона» по
референсу заказчика). Токены — `--sidebar-*` (см. §6). Не путать с полной
тёмной темой: контент, таблицы, карточки, модалки остаются светлыми.

## 5. Типографика

| Роль | Шрифт | Источник | Применение |
|---|---|---|---|
| UI / body / display | **Golos Text** (variable, 400–900) | Google Fonts (cyrillic) | Всё — заголовки, тело, кнопки, таблицы |
| Numbers / коды | **JetBrains Mono** | Google Fonts | Суммы, номера дел, даты-в-таблицах, проценты |

Одна семья гротеска для всего UI + mono для цифр. Никаких serif.

**Глобально:** `font-variant-numeric: tabular-nums`.

| `display-md` | 28 / 1.2 / -0.015em | 600 | Имя клиента в карточке, заголовок страницы |
| `display-sm` | 20 / 1.3 / -0.01em | 600 | Section headings |
| `h-lg` | 18 / 1.35 | 600 | Подсекции, заголовки модалок |

- ❌ Roboto, Arial, Helvetica, system-ui для основных ролей

### Категории дел (3 — цвет несёт % зарплаты)

### Тёмная навигация (sidebar)

Единственная тёмная поверхность системы (решение 2026-05-29). Контраст текста
на тёмном фоне проверен на AA.

| Токен | Hex | Назначение |
|---|---|---|
| `--sidebar-bg` | `#0E1424` | Фон навигации (deep indigo-slate) |
| `--sidebar-elevated` | `#161E33` | Поиск-триггер, плашка пользователя |
| `--sidebar-border` | `#1E2740` | Разделители внутри тёмной зоны |
| `--sidebar-text` | `#94A3B8` | Неактивные пункты |
| `--sidebar-text-strong` | `#F1F5F9` | Логотип, hover, имя пользователя |
| `--sidebar-hover-bg` | `#1B2540` | Hover пункта |
| `--sidebar-active-bg` | `#26315A` | Активный пункт (indigo-tinted) |
| `--sidebar-active-text` | `#A5B4FC` | Текст активного пункта (indigo-300) |

Активный пункт — `bg-active` + indigo-текст + левая полоска-индикатор (3px).
Цвет на сайдбар наносится ТОЛЬКО через эти токены; семантические/этап-цвета
в навигацию не заносятся.

### Запреты

- ❌ Цвет на больших фоновых поверхностях **контента** (страница, header). Sidebar — тёмная навигация по `--sidebar-*` (см. выше), это единственная тёмная поверхность.
- ❌ Градиенты в кнопках и хроме (логотип-плитка в сайдбаре — допустимое исключение, indigo-gradient)
- ❌ Чистые `red` / `green` / `blue` без `bg` пары — всегда работаем парой цвет + light-фон в badges/chips
- ❌ Декоративная заливка карточек

## 9. Границы и тени

- Границы: **1px** solid `var(--border)`. Strong-вариант (`--border-strong`) — для
  hover/focus-state на интерактивах.
- Тени мягкие, с лёгким blue-tint (под холодную палитру):
  - `--shadow-sm`: `0 1px 2px rgba(15, 23, 42, 0.04)` — статичные карточки
  - `--shadow-md`: `0 4px 12px rgba(15, 23, 42, 0.06), 0 1px 2px rgba(15, 23, 42, 0.04)` — поднятые карточки, hover, popover
  - `--shadow-lg`: `0 12px 32px rgba(79, 70, 229, 0.10), 0 2px 6px rgba(15, 23, 42, 0.05)` — модалки (с лёгким indigo-glow)

## 10. Движение

| Токен | Значение | Когда |
|---|---|---|
| `--ease-out` | `cubic-bezier(0.16, 1, 0.3, 1)` | Появление, hover-lift |
| `--ease-in` | `cubic-bezier(0.7, 0, 0.84, 0)` | Уход, collapse |
| `--ease-inout` | `cubic-bezier(0.65, 0, 0.35, 1)` | Reorder, move |
| `--dur-micro` | 80ms | Hover-цвет |
| `--dur-short` | 180ms | Кнопки, инпуты, badge |
| `--dur-medium` | 320ms | Modal, drawer |

Primary-кнопка на hover — `translateY(-1px) + soft indigo shadow` (NetHunt-style lift).

## 12. Компоненты

Базируемся на архитектуре **shadcn/ui** (CVA + Radix primitives + Tailwind), но
имена токенов наши (`primary`/`surface`/`text`), не shadcn-default. Все компоненты
кастомизируются под наши токены при создании.

**База:**
- `Button` — primary (indigo, с hover-lift), secondary (white + border), ghost, destructive
- `Input` — на `surface-muted`, без border по умолчанию, focus = primary border + ring
- `Label`, `Textarea`, `Select`, `Checkbox`, `Radio`
- `Card` + CardHeader/Title/Description/Content/Footer
- `Badge` — status (success/warning/error/info) + priority (low/mid/high) — pill-shaped
- `StageBadge` — chip для этапа дела (5 вариантов)
- `Avatar` — круглый, с инициалами на gradient-фоне (если фото нет)
- `SourceIcon` — мини-иконки коммуникации в брендовых цветах
- `KpiTile` — метрики в карточке дела
- `Dialog`, `Dropdown`, `Tabs`, `Tooltip`, `Toast` — по мере необходимости

**Карточка клиента / лида / дела (NetHunt-style):**
- Hero-шапка — `--grad-indigo`, белый текст, аватар с белой обводкой, semi-translucent action pill
- Секции сворачиваются (chevron справа)
- Внутри секций — Input на `surface-muted` без border

## 13. Командная палитра и горячие клавиши

`Cmd/Ctrl + K` — командная палитра (поиск дел/клиентов/действий).

Базовые шорткаты (закладываем с самого начала):
- `Cmd+K` — палитра
- `Cmd+N` — новое дело
- `Cmd+/` — фокус на поиск
- `Esc` — закрыть модалку
- `?` — справка по шорткатам

---

## 14. Доступность

- WCAG AA минимум.
- Все фокус-стили видимы: primary border + 3px primary-subtle ring.
- Все интерактивные элементы реагируют на клавиатуру.
- `aria-label` для кнопок-иконок.
- Цвет — никогда не единственный носитель смысла (этапы / приоритеты дублируются
  текстом).

## 15. Запреты — сводный список

- ❌ Полная тёмная тема контента (НЕ ДЕЛАЕМ в Phase 1). Тёмный — только sidebar (§4, §6).
- ❌ Inter / Roboto / Arial / Helvetica для основных ролей
- ❌ Serif в display
- ❌ Градиенты на кнопках или больших фонах (логотип-плитка в сайдбаре — исключение)
- ❌ «1С-look»: серый невыразительный dashboard, плотные модалки с табами сверху. NB: premium-тёмный sidebar САМ ПО СЕБЕ разрешён (решение 2026-05-29) — запрещён именно унылый «1С»-стиль, а не тёмная навигация.
- ❌ Default shadcn (slate-палитра, neutral)
- ❌ Чистые `red`/`green`/`blue` без `bg`-пары
- ❌ Centered everything с одинаковыми отступами
- ❌ Bubble radii (rounded-full на всех элементах)
- ❌ Generic stock-hero иллюстрации
