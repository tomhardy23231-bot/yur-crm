## 1. Контекст

- **Что:** CRM для юридической компании; внутренний инструмент на 8 часов в день.
- **Кто:** owner, admin, офис-менеджеры, юристы (продажники), эксперты (исполнители).
- **Класс:** data-tool — плотные таблицы, списки «карточки-строки», формы, kanban-воронка, отчёты.
- **Язык интерфейса:** украинский (дефолт) + русский.
- **Характер:** строгий, корпоративный, спокойный. Цвет работает на семантику
  (этап / категория / приоритет / деньги), хром нейтрален. Не «1С», не generic-админка.

## 2. Типографика

| Роль | Шрифт | Подключение | Применение |
|---|---|---|---|
| UI / body / display | **Geist** (вариативный) | `next/font`, subsets `latin + cyrillic` | Заголовки, тело, кнопки, подписи |
| Числа и идентификаторы | **Geist Mono** (вариативный) | `next/font`, subsets `latin + cyrillic` | KPI-числа (26px bold), суммы, номера дел, клиент-подстроки, время задач/событий, номера дней календаря, `<kbd>`-подсказки |

- Стилистические сеты Geist включены глобально: `font-feature-settings
  "ss01","cv01","cv03"` + `tabular-nums` на `body`.
- **Mono — язык данных** (каркас 2026-07-13): любое число-значение (сумма,
  счётчик, время, №) набирается `font-mono` + `tabular-nums`; крупные KPI —
  mono 26px bold tracking-tight. Заголовки и body-текст — только Geist Sans.
- Второстепенный текст (`text-text-muted`, `text-text-subtle`) автоматически
  получает вес **500** (@layer base в globals.css) — явные `font-*` утилиты его
  перекрывают.
- Базовый размер 14px / 1.55. Заголовки страниц — в топбаре (18px bold
  tracking-tight), не на страницах. Заголовки секций-карточек — 15px semibold.

### Запреты
- ❌ Inter, Roboto, Arial, Helvetica, system-ui для основных ролей.
- ❌ Serif в display (Lora, Newsreader, Fraunces — мимо).
- ❌ Mono для заголовков, body-текста и первичных названий.

## 5. Тени и движение

Тени (тёплый ink-подтон rgba(20,33,27,…), каркас): `--shadow-xs/sm` (статичные
карточки) → `--shadow-md` (hover) → `--shadow-lg` (hover KPI, поповеры) →
`--shadow-pop` (модалки). **Цветные тени:** `--shadow-brand`/`-hover` — синие
CTA и активные Pill-фильтры «парят»; `--shadow-primary-hover` /
`--shadow-destructive-hover` (color-mix) — hover-тени кнопок.

Движение: `--ease-out/in/inout` + `--ease-spring`; `--dur-micro 120ms /
short 200ms / medium 360ms`. Появления: `fade-in-up` (каскад блоков дашборда
через animationDelay), `screen-in`, `sheet-*`, `bar-grow` (полосы воронки);
валидация — `shake`. Всё гасится `prefers-reduced-motion`.

**Правила hover (каркас 2026-07-13):**
- ✅ Lift `-translate-y-px` + `shadow-brand-hover` — CTA-кнопки (primary/destructive).
- ✅ Lift `-translate-y-0.5` + `shadow-md/lg` + `border-primary-border` —
  самостоятельные КАРТОЧКИ-плитки: KPI, карточки канбана, плитки документов.
- ✅ Строки СПИСКОВ внутри карточки-контейнера — hover ТОЛЬКО фоном
  `bg-primary-softer` (без движения и теней).
- ❌ НИКАКИХ infinite-анимаций в длинных списках — у `StageBadge` проп
  `pulse={false}` для списков.

## 6. Компоненты

Архитектура shadcn/ui (CVA + Radix + Tailwind), имена токенов наши. База:

- **Button** — пилюли (`rounded-full`, каркас): `primary` (заливка
  `--primary-hover` + `shadow-brand`, hover-lift + `shadow-brand-hover`),
  `secondary` (белая + бордер, hover синеет: `border-primary-border` +
  `bg-primary-softer`), `ghost` (hover `bg-primary-softer` +
  `text-primary-pressed`), `destructive`; высоты 36/32/40.
- **Input / Textarea / Select** — белое поле (`bg-surface`) + видимая граница
  (`border-border`, hover → `border-strong`), `rounded-control` (12px),
  фокус — синяя граница + кольцо `ring-2 ring-primary/20`. Серую заливку без
  бордера НЕ использовать («1С-look»).
- **Badge** — pill (`rounded-full`), точка ЯРКИМ тоном + тёмный текст
  `*-text` на подложке `*-bg`; тона semantic/prio/primary. Вариант `quiet` —
  точка + тёмный текст без заливки.
- **StageBadge** — пилюля `rounded-chip`: точка ярким тоном этапа + текст
  `*-fg` на подложке `*-bg`. **CategoryBadge** — прямоугольник `rounded-md`
  БЕЗ точки (различие формой, каркас). Пульс точки — только у «живого» этапа
  и не в списках.
- **Pill-фильтры** (пресеты /cases, сохранённые виды, типы клиентов) — пилюли:
  активный — `bg-primary-hover text-white shadow-brand` (тёмно-синяя заливка с
  цветной тенью), неактивный — белый с бордером, синеет на hover.
- **Сегмент-контрол** (Список/Доска, Активные/Архив, Месяц/Неделя, Мои/Все) —
  контейнер `rounded-xl border bg-surface p-0.5`, активный пункт —
  `rounded-lg bg-primary-subtle text-primary-pressed` (синий тинт).
- **Списки** (`CardListShell` + `ClickableCard`) — ОДНА карточка-контейнер
  (`rounded-card` + `shadow-sm`), шапка колонок на `bg-surface-sunken`
  (sticky, caps 11px semibold `text-text-muted`), строки разделены
  `border-border/60`, hover строки — `bg-primary-softer`. Первичная строка:
  название semibold + mono-подстрока (№/клиент) `text-[11.5px]`.
- **Hero-баннер дашборда** (`dashboard/hero-banner.tsx`) — `rounded-3xl`,
  фон `--grad-hero`, декор-орбы + сетка-узор, приветствие по времени суток
  (Киев) 28–32px bold, дата-чип `bg-white/15`, сводка дня с жирными числами,
  до двух мини-статов на стекле (`bg-white/12 backdrop-blur`, mono-значения).
- **KPI-плитка** (`dashboard/kpi-card.tsx`) — label 12.5 medium + иконка в
  тинт-квадрате 8×8 `rounded-lg`, число mono 26px bold, спарклайн с area-
  градиентом, дельта-чип 11px bold; hover — lift + `shadow-lg`.
- **Канбан** — заголовок колонки: точка этапа + название + счётчик-пилюля
  `bg-surface-sunken`; контейнер `rounded-card bg-surface-sunken/50 p-2.5`
  без бордера; карточки `rounded-xl` с hover-lift.
- **Календарь** — плитки дней `rounded-xl border` с зазором `gap-1.5/2` в
  карточке; сегодня — `border-primary bg-primary-subtle`, выбранный —
  `ring-2 ring-primary/40`; вне месяца — `opacity-55`; события — чипы
  `rounded-md text-[10px]` (подложка типа + точка + название), «+N ещё»;
  заголовки Сб/Вс — `text-error`; номер дня — mono.
- **Table** — плотная: ряд 44px (`h-11`), шапка 40px caps `text-text-subtle`
  (на белом — AA), `sticky` thead, `border-separate` + границы на ячейках
  (границы едут вместе со sticky-шапкой). Числа — вправо, `tabular-nums`.
- **Modal / ConfirmDialog** — `rounded-modal`, подложка `bg-overlay` +
  backdrop-blur, фокус-трап, Esc; деструктив — красная CTA.
- **Avatar** — круг (или `square` 8px для таблиц), инициалы на `--avatar-N`.
- **Командная палитра** `Cmd/Ctrl+K`; kbd-подсказки — JetBrains Mono.
- **Toast** (`ui/toast.tsx`, v3 s11) — обратная связь на действие.
  `useToast().success/error`; для форм, чьи actions делают redirect
  (client-form/case-form) — `flashToast()` через sessionStorage (TTL 15 с,
  показывает провайдер на новой странице). Позиция: desktop — правый низ
  (`bottom-4 right-4`), mobile — по центру над bottom-nav; `z-[110]` (выше
  модалок). Карточка: `bg-surface` + бордер + `rounded-card` + `--shadow-lg`,
  иконка 16px (`CheckCircle2`/`AlertCircle` в success/error-тонах), текст 13px.
  Максимум 3 одновременно, auto-dismiss 4 с с паузой при hover. Появление —
  `animate-toast-in` (fade+slide 150ms, гасится reduced-motion). A11y:
  контейнер `aria-live="polite"`; success → `role="status"`, error → `"alert"`.
  Когда что: success — создание/сохранение сущности (платёж, акт, задача,
  отпуск, «Сохранено» после redirect-форм); error — провал action'а, где нет
  инлайн-баннера формы. `?error=`-баннеры страниц тосты НЕ заменяют.
- **EmptyState** (`ui/empty-state.tsx`, v3 s11) — единый вид «пусто»:
  опц. иконка 28px `text-text-subtle`, title 14px medium, hint 13px muted,
  опц. CTA. Без собственной рамки — родитель оборачивает (Card-секция как
  есть; страницы списков — карточка `rounded-lg border bg-surface py-8`).
  У списков ДВА текста: «пусто вообще» (с CTA) и «не найдено по фильтрам».

## 7. Доступность (WCAG AA — обязательна)

- Обычный текст ≥ 4.5:1, крупный/декоративный ≥ 3.0:1. Проверять скриптом
  (PLAN-V3 Сессия 10 — `contrast.js`), не на глаз.
- Залитые чипы/бейджи — ВСЕГДА пара «подложка `*-bg` + тёмный текст
  `*-fg`/`*-text`». Яркий тон на подложке — только точка/иконка.
- На paper-фоне (`--bg`) вторичный текст — `text-text-muted`
  (`text-text-subtle` там не проходит).
- Фокус видим всегда: `outline 2px var(--primary)` (в сайдбаре —
  `--sidebar-accent-bright`).
- Цвет — не единственный носитель смысла: этап/категория/приоритет дублируются
  текстом, статусы — подписями.
- Клавиатура: все интерактивы фокусируемы, `aria-label` у кнопок-иконок,
  модалки с фокус-трапом и Esc.
