# Тесты Юр CRM

Три уровня. Чем выше — тем быстрее и дешевле; чем ниже — тем ближе к реальному
поведению пользователя.

| Уровень | Папка | Раннер | Что проверяет | Нужна инфраструктура |
|---|---|---|---|---|
| **Юнит** | `tests/unit` | Vitest | Чистая логика (даты ЗП, diff журнала, форматтеры, маппинг ошибок) | нет |
| **Интеграция** | `tests/integration` | Vitest | RLS (права доступа), триггеры денег, ставки/расчёт ЗП, воронка этапов | локальный Supabase |
| **E2E** | `tests/e2e` | Playwright | Реальный браузер: вход, защита маршрутов, app-shell | dev-сервер + Supabase + сид |

## Команды

```bash
npm test                 # юнит-тесты (быстро, без БД) — дефолт
npm run test:unit        # то же
npm run test:watch       # юнит в watch-режиме
npm run test:integration # интеграционные (нужен поднятый Supabase)
npm run test:e2e         # E2E (Playwright сам поднимет dev-сервер на :3210)
```

## Предусловия

**Интеграция и E2E** требуют поднятого локального Supabase и заполненного `.env.local`:

```bash
npx supabase start       # Postgres + Auth + Storage
npm run db:seed          # тестовые аккаунты и данные
```

- Интеграционные тесты **самодостаточны**: создают свой namespace (`IT-<runId>-…`)
  через `service_role`, проверяют от лица обычных сессий и **убирают за собой**.
  Если переменных Supabase в `.env.local` нет — набор помечает себя `skipped`,
  а не падает.
- E2E логинится засеянным владельцем (`owner@yur.local` / `test12345!`, см.
  `scripts/seed.ts`). Первый прогон медленнее — Turbopack компилирует маршруты
  на лету (таймауты в `playwright.config.ts` это учитывают).
- Перед E2E один раз поставить браузер: `npx playwright install chromium`.

## Что НЕ покрыто (кандидаты на расширение)

- Server Actions создания/правки (клиент, дело, платёж) — позитив + отказ по правам.
- E2E-сценарии создания клиента → дела → платежа и отчёт по ЗП.
- Идемпотентность платежей (`idempotency_key`, 23505), удаление документов.

Смежно: `scripts/smoke-rls.ts` (`npm run smoke:rls`) — широкий смоук RLS/триггеров
после миграций (исторический, пересекается с `tests/integration`).
