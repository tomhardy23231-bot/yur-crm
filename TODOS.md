## Цикл «Корпоративный сервер» (после v4)

- **Что:** перенести CRM с облаков (Vercel + Neon + R2) на собственный сервер
  компании: Docker/standalone-сборка Next.js, PostgreSQL на сервере,
  файлы — MinIO или локальный диск (реализация за интерфейсом
  `lib/storage.ts`), почта — корпоративный SMTP (env-переменные nodemailer).
- **Зачем:** решение владельца (2026-07-14): под CRM специально купят
  корпоративный сервер; юридические данные — на своём железе.
- **Контекст:** цикл v4 (docs/PLAN-V4-POSTGRES.md, §1 п.4) сознательно выбирал
  компоненты без vendor lock-in: БД — обычный Postgres (перенос = pg_dump +
  смена DATABASE_URL), файлы — S3-совместимый интерфейс (R2 → MinIO без правки
  вызовов), auth — свой (переезжает с приложением), почта — любой SMTP.
  Начинать с: инвентаризация сервера (ОС, докер, домен, TLS), затем
  docker-compose (app + postgres + minio + бэкапы), затем репетиция переноса
  данных с Neon. Отдельно решить: мониторинг и автобэкапы на своём железе
  (в облаке это давали Neon/Vercel).
- **Зависит от:** завершения цикла v4 (переезд на Neon) и покупки сервера.
